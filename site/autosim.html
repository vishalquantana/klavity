Why AutoSim is different

Self-healing that never hides a real bug.

Most test automation pays for maintenance with false confidence. AutoSim flips it: heal the noise, surface the signal, and stay cheap enough to run on every commit.

heal, don't hide

Intent-verified healing

When a selector breaks, the heal ladder re-finds the element by meaning — cache → role + accessible name → vision. It's diagnosis-first: it heals only when the goal still completes, so a real regression is filed, never papered over.

free on every build

Zero-LLM deterministic replay

Unchanged UI replays from the locator cache with no model calls at all — repeatable and cheap enough to gate every release. AI is spent only to author, heal, or diagnose, not on the happy path.

grounded in people

Persona-driven correctness

Your Sims — personas built from real customer calls — are the oracle. A run isn't "did the script pass?" but "could this person actually finish what they came to do?"

no lock-in

Open-core & Playwright-exportable

Every flow crystallizes into a real, ownable Playwright test plus its locator cache. AutoSim is open-core — keep the tests, export them, run them anywhere. The engine earns its keep; it doesn't trap you.

Questions

The fine print on self-healing tests.

How does AutoSim self-heal a broken test?

When a selector breaks, AutoSim re-finds the element by meaning using a heal ladder — locator cache, then role + accessible name, then vision — and keeps going, flagging the change as an amber heal-diff (e.g. #checkout → #pay-now-btn) instead of failing.

Will self-healing hide a real bug?

No. Healing is intent-verified and diagnosis-first. If the goal genuinely can't complete, AutoSim files a grounded finding rather than quietly passing green — so it never produces a silent false-green.

How much does it cost to run on every build?

Unchanged UI replays deterministically with zero LLM calls, making it cheap and fast enough to gate every release. AI is only spent to author, heal, or diagnose.

Is Klavity AutoSim open and exportable?

Yes. AutoSim is open-core and crystallizes each flow into a real, ownable Playwright test plus a locator cache — so there's no lock-in.
