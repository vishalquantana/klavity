The cloud flow

5
Tickets flow to your account. The destination banner makes it explicit — filing to Acme on Klavity Cloud.
6
Recent tickets, in the popup. See what you (and your Sims) filed, with live status — no context-switch.

Big, obvious actions. Bug / Feature front-and-centre, with the shortcut + right-click hint.

Optional mirror. Quietly push a copy to Jira/Linear too, shown in the footer.
This is a static mock. It already maps onto the real model: cloud submits go through core/integrations/backend.ts → POST {backendUrl}/api/feedback. To make it live we add a Klavity account token (sign-in → token in chrome.storage.session, refresh in local), default backendUrl to Klavity Cloud, and the backend associates feedback with the account + returns it for the “Recent” list. The greeter Sim uses the real @klavity/core/sim component (/klavity-sim.js).
