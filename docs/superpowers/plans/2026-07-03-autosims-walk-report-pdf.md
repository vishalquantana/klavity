# AutoSims Walk Report PDF Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** A branded, shareable Walk Report — "Export PDF" on any Walk plus an unguessable expiring share link that serves the same PDF without login.

**Architecture:** The runner gains opt-in per-step screenshot capture (S3 via existing `lib/s3.ts`, key in `run_steps.evidence.screenshotKey`). A pure renderer (`lib/trails-report.ts`) turns a Walk's data into a standalone branded HTML document; server-side Chromium (`page.setContent` + `page.pdf`, guarded by the walk slot) prints it. Share links are hashed tokens in a new `walk_share_tokens` table.

**Tech Stack:** Bun, libsql, Playwright chromium (already on the box), existing S3 helpers (`uploadScreenshot`, `presignGet`), self-hosted Fraunces/Hanken fonts (`site/fonts/`), plain-HTML UI.

## Global Constraints

- Branding: Klavity wordmark + indigo accent (`#6366f1`-family tokens as used in trails.html), footer "Generated by Klavity AutoSims · klavity.in"; Fraunces for display, Hanken Grotesk for body (absolute URLs `https://klavity.in/fonts/...` in the report HTML so Chromium can load them; renderer takes a `baseUrl` param — tests pass a dummy).
- ADR-0001 still binds: report shows `{{cred:...}}` placeholders verbatim, never secrets; per-step screenshots are captured AFTER actions settle (password fields render dotted — acceptable).
- Share tokens: 32-byte random hex, stored SHA-256-hashed (`sha256hex` from lib/crypto.ts), project-scoped, default expiry 30 days, single walk each. Route serves PDF only — no session, no cookies. 404 (not 401) on bad/expired token.
- PDF generation launches Chromium on the shared 1GB box: MUST hold the walk slot (`withWalkSlot`) → 409 "AutoSim busy" if walking; `CHROMIUM_PROD_ARGS`; PDF gen bounded by a 30s deadline; injectable renderer in route tests (no chromium in route tests; ONE real-chromium pdf e2e in the lib test).
- Per-step screenshots: opt-in `WalkOptions.stepShots?: boolean` (default OFF; the manual walk trigger `realWalk` passes true). jpeg quality 45. Injectable `shotUploader` (fake in tests — never S3). Upload failures are best-effort: never fail or slow a step (try/catch, evidence just lacks the key).
- Old walks without screenshots: report renders gracefully (steps table without images).
- Straight quotes only in HTML/JS; `node scripts/check-inline-js.mjs` must pass; no emoji (kicon/inline SVG only); micro-animation tokens for new buttons; hermetic bun:tests; no version bumps.
- Worktree `klav-snap-wt-autosims-domain-model`, branch `feat/autosims-domain-model`; run from `prototype/` unless noted.

### Task 3: PDF route + share tokens

**Files:** Modify `prototype/server.ts` (inside the trails auth gate + one NEW unauthenticated route), `prototype/lib/db.ts` (schema), create `prototype/lib/trails-share.ts`. Test: `prototype/server.walk-report.route.test.ts`.

**Schema:** `CREATE TABLE IF NOT EXISTS walk_share_tokens (id TEXT PRIMARY KEY, token_hash TEXT NOT NULL UNIQUE, run_id TEXT NOT NULL, project_id TEXT NOT NULL, created_by TEXT, expires_at INTEGER NOT NULL, created_at INTEGER NOT NULL)` + index on token_hash.

**Interfaces (`lib/trails-share.ts`):** `mintShareToken(projectId, runId, createdBy?, ttlMs = 30*24*3600e3): Promise<string>` (returns the RAW token once); `resolveShareToken(rawToken): Promise<{ projectId, runId } | null>` (hash lookup + expiry check); `renderWalkPdf(projectId, runId, baseUrl): Promise<Uint8Array>` — gather → render → withWalkSlot(chromium setContent+pdf, 30s deadline, CHROMIUM_PROD_ARGS) — with an injectable module-level override for tests (`export function _setPdfRendererForTests(fn)` pattern is acceptable here, mirroring existing injectable seams).

**Routes:**
- `GET /api/trails/walks/:runId/report.pdf` (inside the trails gate; extend the gate's walks path match if needed) → 404 unknown/cross-project; 409 WalkBusyError; else `application/pdf`, `content-disposition: attachment; filename="klavity-walk-<shortid>.pdf"`.
- `POST /api/trails/walks/:runId/share` (authed, same gate) → `{ url: BASE + "/shared/walk-report/" + rawToken, expiresAt }`.
- `GET /shared/walk-report/:token` (UNAUTHENTICATED, register near other public GETs) → resolve token → 404 bad/expired → else same PDF (inline, not attachment).
